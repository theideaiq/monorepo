## 2025-04-08 - Unhandled Date Parsing Errors in format.ts
Discovery: `formatDate` in `packages/utils/src/format.ts` only validated `date.getTime()` if the argument was explicitly a `Date` instance. When passed an invalid string like `"invalid"`, it parsed it on the fly and crashed with `RangeError: Invalid time value` because there was no validation for the parsed date. This reveals a risk where shared generic utility functions can crash the entire page or app if they assume correct string formats without boundary checks.
Strategy: Use Boundary Analysis and Unhappy Paths for generic utility functions. Always pass edge-case inputs like empty strings, nulls, and `"invalid"` representations to formatting functions and verify they return fallback UI strings (like `""`) instead of throwing unhandled exceptions.
