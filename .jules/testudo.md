## 2025-04-03 - [Invalid Date Object Crash Prevention]
Discovery: [Intl.DateTimeFormat throws a `RangeError` if initialized with an `Invalid Date` object generated by `new Date('invalid')`. Simple `!date` checks are insufficient because `new Date(string)` returns an object whose `getTime()` is NaN, which bypasses truthy checks.]
Strategy: [Always explicitly validate date inputs by calling `.getTime()` and checking for `Number.isNaN()` before passing them to the `Intl` API or formatting functions.]
