## 2025-02-13 - Cache Intl.NumberFormat and Intl.DateTimeFormat
**Learning:** Instantiating `Intl.NumberFormat` and `Intl.DateTimeFormat` on every React render or inside tight loops causes significant memory allocation and CPU overhead, especially with complex lists or charts. This was discovered to be a bottleneck when generating formatters dynamically within React render paths across the admin and web applications.
**Action:** When caching `Intl.NumberFormat` or `Intl.DateTimeFormat` instances, using `JSON.stringify(options)` to generate a cache key is safe and acceptable because the formatting options objects are stable and small. Always use the cached `getNumberFormatter` and `getDateTimeFormatter` utilities from `@repo/utils` to reduce object allocation and CPU overhead instead of direct `new Intl.NumberFormat` or `new Intl.DateTimeFormat` instantiation.
