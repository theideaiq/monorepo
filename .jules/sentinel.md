## 2026-04-05 - Insecure checkout of head.sha in pull_request_target workflows
**Vulnerability:** Checking out `github.event.pull_request.head.sha` in a workflow triggered by `pull_request_target`.
**Learning:** `pull_request_target` workflows run with elevated privileges (like access to secrets and write tokens) meant for the base repository. When you explicitly check out the code from the untrusted PR fork (`head.sha`), the workflow executes against that untrusted codebase. In the case of `actions/labeler`, the malicious contributor can spoof configuration files to grant themselves auto-approval labels or other unauthorized tags. If scripts are run, it can lead to full Remote Code Execution (RCE).
**Prevention:** Never check out the untrusted fork (`head.sha`) in a `pull_request_target` event unless you are absolutely certain no arbitrary code or spoofable configurations can be executed. Rely on the base branch repository (`main`) configurations whenever running actions with elevated privileges.
